# freee アプリの権限（TASUKE-AI 用）

開発者コンソール（freee アプリ設定）で、次を **有効** にしてください。  
**変更後は必ず再認可**し、`node freee-auth.js token <認可コード>` でトークンを取り直してください。

## レシート取込（`receipt-import.js`）
| 種別 | 内容 |
|------|------|
| 事業所 | 参照 |
| 取引 | 参照・登録・更新（学習・更新用） |
| 取引先 | **参照・登録・更新**（新規店の自動作成に必須） |
| 勘定科目・**品目**・部門 | 参照（**品目マスタを読み、明細に必ず品目を付ける**） |
| 口座・現金・カード等 | 参照（支払元の指定） |

## 売上・学習（`freee-sales.js` / `receipt-import.js --learn`）
| 種別 | 内容 |
|------|------|
| 事業所 | 参照 |
| 取引 | 参照（過去取引の取得） |
| 取引先 | 参照 |
| 勘定科目・品目・部門 | 参照 |
| 損益計算書・試算表など | 参照（`--pl` 利用時） |

※ 表示名は freee のアプリ画面のラベルと多少異なる場合があります。  
「取引」「取引先」「マスタ」の **読み取り＋作成（＋更新）** が通るようにしておくのが安全です。
