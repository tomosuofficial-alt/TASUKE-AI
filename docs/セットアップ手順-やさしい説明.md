# セットアップ手順（やさしい説明）

難しい言葉は少なめです。上から順にやってください。

---

## まず知っておくこと

### GitHub の「Secrets（シークレット）」って何？

パスワードやトークンみたいな **秘密の文字列** を、GitHub に預けておく場所です。

- **なぜ使う？**  
  プログラムが Notion や freee にアクセスするときに必要だから。  
  でもその秘密を **ソースコードに書くとバレる** ので、GitHub の「金庫」に入れます。

### いまの状態（このプロジェクト）

- **Notion の案件管理DB** に、もう **「請求ステータス」** という列は **追加済み** です（未請求／請求済／不要）。
- あなたがやるのは主に **GitHub に秘密を登録する作業** と、Notion で **各行に請求ステータスを選ぶ作業** です。

---

## A. GitHub に秘密を登録する（パソコンでブラウザ）

### 準備

1. ブラウザで **自分のこのプロジェクトの GitHub ページ** を開く。  
2. 画面上の **「Settings」（設定）** をクリック。  
   （リポジトリのトップでは、上の方のメニューにあることが多いです。）

### Secrets を開く

3. 左のメニューを下にスクロールして **「Secrets and variables」** を探す。  
4. その中の **「Actions」** をクリック。  
5. **「New repository secret」**（新しいシークレット）みたいなボタンを押す。

### 1つずつ登録する

**ルール：**  
- **Name（名前）** は **英語のまま、スペルもそのまま**（コピペ推奨）。  
- **Secret（中身）** は、下の表の「中身の作り方」どおり。

| 名前（Name にコピペ） | 中身は何？ |
|----------------------|------------|
| `NOTION_TOKEN` | Notion の「インテグレーション」の秘密の文字列（長いやつ）。すでに `.env` にあればそれと同じ。 |
| `FREEE_CLIENT_ID` | freee のアプリ画面にある Client ID。`.env` の `FREEE_CLIENT_ID` と同じ。 |
| `FREEE_CLIENT_SECRET` | freee のアプリの Client Secret。`.env` と同じ。 |
| `FREEE_TOKEN_JSON` | 下の「B」をやって出てきた **1行の長い文字**（一番むずかしい）。 |
| `SLACK_DAILY_COMMAND_WEBHOOK_URL` | Slack の Webhook の URL。`.env` と同じ。（朝の通知などに使うワークフロー用） |

**やり方（毎回同じ）：**

1. **Name** に上の表の左の文字をコピペ。  
2. **Secret** に中身を貼る。  
3. **保存** ボタンを押す。  
4. 次の名前でもう一度。

---

## B. `FREEE_TOKEN_JSON` だけ特別な作り方

これは **「1行にまとまった JSON」** です。手元の Mac でターミナルを開いてください。

### 手順

1. プロジェクトのフォルダに移動する（いつも `npm` を打っている場所）。  
2. 次を **この順番** で打つ。

```bash
node freee-auth.js refresh
```

（freee とつながるための更新。うまくいくと「成功」みたいな表示が出る。）

3. 次を打つ。

```bash
node scripts/print-freee-token-json-for-github-secret.js | pbcopy
```

- 画面にはあまり何も出ないかもしれないが、**中身はクリップボードにコピーされている**。  
4. GitHub の **New secret** で、名前を **`FREEE_TOKEN_JSON`** にして、**貼り付け（Command + V）** して保存。

**注意：** この中身も **誰にも見せない・チャットに貼らない**。

---

## C. Notion で「請求ステータス」を使う

1. **案件管理** のデータベースを開く。  
2. 表に **「請求ステータス」** という列があるはず。  
3. 各行で、次のどれかを **選ぶ**。

| 選ぶ言葉 | いつ使うか（イメージ） |
|----------|-------------------------|
| **未請求** | まだ請求書を出していない／出す予定がある |
| **請求済** | もう請求した |
| **不要** | この案件は請求しない（無料・社内など） |

4. 選び終わると、**月初の未請求アラート** は「未請求」だけをしっかり拾いやすくなります。

（まだ全部埋めなくても動きます。空の行は、昔のルールで「要確認候補」に出ることがあります。）

---

## D. うまくいったか試す（おまけ）

1. GitHub の **「Actions」** タブを開く。  
2. 左から **「Monthly freee KPI sync」** を選ぶ。  
3. **「Run workflow」** で手動実行。  
4. 緑のチェックならだいたい成功。赤ならログを開いて、英語のエラーを検索したり、トークンをもう一度更新したりする。

---

## 困ったとき

- **「Settings が見えない」**  
  → そのリポジトリの **管理者** じゃない可能性があります。オーナーに「Secrets を登録したい」と言ってください。  
- **`FREEE_TOKEN_JSON` で失敗する**  
  → もう一度 `node freee-auth.js refresh` してから、もう一度 B のスクリプトでコピーし直し、GitHub の Secret を **上書き更新** してください。

もっと詳しい一覧は **`docs/setup-github-secrets.md`**（大人向け・表が多い）にあります。
