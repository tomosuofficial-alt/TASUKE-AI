# Notion で見やすくする（やさしい説明）

コンテンツ DB 用。

---

## まずここだけ（結論）

**Notion の表は、Excel のように「ここからここまでが企画ブロック」と線で囲めません。**

だから **同じデータベースを、「見る画面」を2つに分ける**のがおすすめです。

| 見る画面（ビュー）の名前 | 何をするか |
|--------------------------|------------|
| **いつもの編集** | キャプションや素材など、**よく触る列だけ**出す（数字の列は隠す） |
| **数字だけ** | リーチ・いいねなど、**投稿あとに入る数字の列**だけ出す |

**やることは1つ。**  
表の上にある **「Default」みたいな名前**をクリック → **ビューを複製** → 名前を変える → **列の「目」アイコンで、見せたい列だけオン**。

これで「ブロック分けしたような」見え方になります。むずかしい操作はありません。

---

## なぜこんなことをするの？

- **全部の列を一度に出すと**、横に長くて見づらい。
- **編集するとき**は、キャプションやフックなど **文字の列**が大事。
- **あとから見る数字**（リーチなど）は、**まだ空のことが多い**ので、普段は隠してよい。

→ **「編集用の画面」と「数字用の画面」を分ける**と、どちらも見やすい。

---

## 手順（クリックだけ）

1. コンテンツの **データベース**を開く（表が出ている画面）。
2. 表の **上の左あたり**にある **ビュー名**（例: `Default` や `表`）をクリック。
3. **「複製」** または **「ビューの複製」** を選ぶ。
4. 新しくできたビューの名前を、例えば **`いつもの編集`** に変える。
5. 表の **右端の「+」の左**、または **「プロパティ」** を開く。
6. **表示したい列だけオン**にする。  
   - **`いつもの編集`** なら：タイトル、日付、ステータス、キャプション、ハッシュタグ、コピペ用…など **文字まわり中心**。リーチ・いいねなどの **数字はオフ**でOK。
7. もう一度 **ビューを複製**して、名前を **`数字だけ`** などにする。
8. 今度は **タイトル・投稿日・クライアント** だけオンにして、あとは **リーチ・いいね・保存…** など **数字の列だけオン**。

あとは、**普段は「いつもの編集」を開く**だけで大丈夫です。

---

## 「親ページに2つ並べる」は（お好みで）

もっとページっぽく分けたいときだけでOKです。

1. 空のページを新規作成。
2. 見出しを書く：例 **「編集」**
3. `/リンク` と打つ → **データベースをリンク** → コンテンツ DB を選ぶ。
4. 出てきた表の右上 **⋯** で、さっき作った **`いつもの編集`** ビューを選ぶ。
5. その下に見出し **「数字」** を書いて、もう一度 `/リンク` で同じ DB を出し、**`数字だけ`** ビューを選ぶ。

**やらなくても大丈夫。** ビューが2つあれば、上の「手順だけ」で十分です。

---

## 用語メモ（読み飛ばしOK）

| 言葉 | 意味 |
|------|------|
| **ビュー** | 同じデータベースを、「どの列を見せるか」違いで保存した見方。切り替えタブのようなもの。 |
| **リンクドビュー** | 別のページに、その DB の一部を貼り付けたもの。お好みで。 |

---

## 自動ではできないこと

プログラム（API）から **列の並び**や **ビュー**をこちらで勝手に作ることは、ほぼできません。**Notion アプリで手でやる**必要があります。
