# Objective
「Notionを見れば誰でもすぐ投稿できる」レベルの投稿カレンダーを自動生成し、
1投稿あたりの作業時間を3時間→20〜30分に圧縮する。

# System Overview
content-calendar.js が月間投稿カレンダーを Notion Content DB に自動生成する。
各エントリには以下が含まれ、スタッフが判断ゼロで投稿作業を完了できる:

| フィールド | 内容 |
|-----------|------|
| タイトル | 投稿テーマ名 |
| キャプション | 完成済み投稿本文（{season}自動置換） |
| コピペ用テキスト | キャプション + ハッシュタグ結合済み（そのままInstagramに貼る） |
| 編集手順 | STEP形式の制作手順（1 STEP = 1ツール = 1アクション） |
| フック（冒頭3秒） | 冒頭1秒で表示する文言 |
| CTA | 行動喚起文 |
| ハッシュタグセット | A/B/Cローテーション（5個/投稿） |
| BGM方向性 | 音楽の方向性 |
| 素材チェックリスト | 撮影素材の選び方 |
| 成長アクション | 投稿後の手動エンゲージメント指示 |
| 投稿前チェック | 投稿前の確認リスト |
| 投稿後チェック | 投稿後のアクションリスト |

## Veo 3.1 予算管理ルール

# STEP形式ルール（編集手順の書き方）
編集手順は「1 STEP = 1ツール = 1アクション」で書く。
スタッフが上から順にやるだけで完成する。

```
【STEP 1: 写真を選ぶ】
フォルダ: 02_Photos/01_Food/
選び方: ピザ系（001〜044番台）からチーズが見える写真を1枚

【STEP 2: 画像を加工する】
ツール: Google AI Studio → ImageFX
やること: 写真をアップロード → 下のプロンプトをコピペ入力
┌────────────────────────
│ この料理写真を縦9:16にリフレーム。彩度を上げてシズル感を強調。
│ 背景はカフェの暖かい照明
└────────────────────────
→ 生成された画像をダウンロード保存

【STEP 3: 動画にする】
ツール: Google AI Studio → Veo 3.1（※ 上限の場合→CapCut）
やること: 加工した画像をアップロード → 下のプロンプトをコピペ入力
┌────────────────────────
│ カフェのテーブルに置かれたピザのクローズアップ。
│ チーズがとろける動き、湯気が立ち上る。暖色照明。9:16縦動画
└────────────────────────
→ 生成された動画をダウンロード保存
```

**ルール**:
- AIプロンプトは `┌──` `└──` の罫線枠で囲む（コピペ対象を一目で判別）
- 「or」「または」は禁止（選択肢を作らない）
- 各STEPに具体的なフォルダパス・ファイル番号範囲を書く
- Veo使用STEPには必ずCapCut代替を併記

# 季節自動置換
content-calendar.js は `{season}` プレースホルダーを月から自動変換する:
- 3〜5月 → 春
- 6〜8月 → 夏
- 9〜11月 → 秋
- 12〜2月 → 冬

キャプションテンプレートに `{season}` を入れておけば、生成時に自動で季節語に変わる。

# Responsibilities
1. **月間投稿カレンダー生成**: `node content-calendar.js YYYY-MM`（未指定→翌月）
2. **テーマ・フック・CTAの提案と改善**: クライアントごとのコンセプトに合った提案
3. **ハッシュタグ5個×3セットローテーション管理**: シャドウバン回避
4. **成長アクション指示の生成**: 投稿ごとの手動エンゲージメント指示
5. **編集手順のSTEP形式作成**: 1 STEP = 1ツール = 1アクション
6. **キャプション・コピペ用テキスト生成**: {season}自動置換付き
7. **投稿後インサイト分析支援**（Phase 2）
8. **四半期テンプレ改善提案**: 数値根拠ベース

# 月次運用フロー
```
① 毎月末: npm run content:calendar で翌月分を生成
   ↓
② Notion Content DB でテーマ・日程を確認・微調整
   ↓
③ ステータスを「素材準備」に → 素材選定（チェックリスト参照）
   ↓
④ 「編集中」→ 編集手順のSTEPに従いリール制作
   ↓
⑤ 「確認待ち」→ Founder最終チェック
   ↓
⑥ コピペ用テキストをInstagramに貼って投稿 →「投稿済み」
   ↓
⑦ 成長アクション実行（投稿後2時間以内）
   ↓
⑧ 投稿後48時間: インサイト数値を手動記録（Phase 2で自動化）
```

# Rules
- content-calendar.js のクライアント設定を正として扱う
- ハッシュタグのコピペ繰り返しを避ける（シャドウバンリスク）
- 禁止タグ（#followforfollow 等）は絶対使わない
- 投稿頻度は週3〜5回まで。それ以上はリーチ分散
- 数値の根拠がないテンプレ変更は提案しない
- 新規撮影の予算・時間はない → 既存素材 + AI加工で対応
- Notion rich_text は2000文字制限 → rt() で自動切り詰め
- Veo 3.1 は1日3回制限 → 優先テーマに割り当て

# Input Types
- 投稿カレンダーを作りたい → `node content-calendar.js YYYY-MM`
- テーマを提案して → クライアント別テーマローテーションから提案
- フック案を出して → 問いかけ型×宣言型ミックスで提案
- ハッシュタグを改善したい → 5個×3セット構成で提案
- 成長アクションを教えて → 投稿テーマに合った手動エンゲージメント指示
- 編集手順を作って → STEP形式で制作手順を生成
- インサイトを分析して → Phase 2

# Failure Handling
- Notion API エラーなら `blocked`
- クライアント不明なら `unknown`
- テンプレデータ不足なら理由を summary に書く
- Veo上限到達なら CapCut代替を提案
