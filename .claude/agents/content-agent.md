# Objective
「Notionを見れば誰でもすぐ投稿できる」レベルの投稿カレンダーを自動生成し、
1投稿あたりの作業時間を3時間→20〜30分に圧縮する。

# System Overview
content-calendar.js が月間投稿カレンダーを Notion Content DB に自動生成する。
各エントリには以下が含まれ、スタッフが判断ゼロで投稿作業を完了できる:

| フィールド | 内容 |
|-----------|------|
| タイトル | 投稿テーマ名 |
| キャプション | 完成済み投稿本文（{season}自動置換） |
| コピペ用 | キャプション + ハッシュタグ結合済み（そのままInstagramに貼る） |
| 作業手順 | スタッフが手を動かすSTEP（ミセル自動処理は含めない） |
| 冒頭フック | 冒頭1秒で表示する文言 |
| CTA | 行動喚起文 |
| ハッシュタグ | A/B/Cローテーション（5個/投稿） |
| BGM方向性 | 音楽の方向性 |
| 使用素材 | 使う素材ファイルの一覧（ミセルが自動選定） |
| AI出力先 | AI生成済み画像のパス（ミセルが自動記録） |
| 成長アクション | 投稿後の手動エンゲージメント指示 |
| 投稿前チェック | 投稿前の確認リスト |
| 投稿後チェック | 投稿後のアクションリスト |

# DaVinci Resolve テンプレート運用

## 基本方針
DaVinci Resolve（無料版）のプロジェクトをテンプレートとして運用する。ゼロから編集しない。
プロジェクトを複製 → 素材を差し替え → テキストを差し替え → デリバー（書き出し）。
1本あたりの作業時間の目標: **5〜10分**。

## なぜDaVinci Resolveか
- 無料版でもトランジション50種類以上・エフェクト・カラグレが全て使える
- CapCut無課金だとトランジションがほぼ「スライド」のみで制限が厳しい
- Ken Burns（ダイナミックズーム）が標準搭載
- プロジェクト複製でテンプレート運用が可能
- BGMは別途フリー音源を用意（Pixabay Music, Artlist Free 等）

## BGM運用
DaVinci ResolveにはBGMライブラリがないため、フリー音源を事前にダウンロードしておく:
- **M'z cafe用**: Pixabay Music で `cafe jazz` `lounge` 検索 → 15秒以上の曲を1曲DL
- **Niki★DINER用**: Pixabay Music で `upbeat pop` `energetic` 検索 → 15秒以上の曲を1曲DL
- 保存先: 各クライアントの素材フォルダ内 `04_BGM/` に保存
- テンプレートに固定で読み込んでおく（毎回検索しない）

## テンプレート一覧（3種類）

全テンプレート共通構成: **動画1本（7.5s）→ 画像3枚（各2.5s）**（合計15秒）
動画で目を止め、画像で情報を伝え、CTAで締める「掴み→説得→行動」構造。

# 画像生成ルール

## 基本方針
- 保存先: 各クライアントの `04_AI_Generated/` フォルダ
- ファイル名: `YYYYMMDD_テーマ名_番号.png`（例: `20260407_調理ライブ_1.png`）
- 形式: PNG
- サイズ: 1080x1920（9:16）
- Notionの「AI出力先」にパスを記録し、ステータスを「AI生成済み」に更新

## 素材が不足・不適合の場合
キャプションにふさわしい画像素材がない場合は、既存素材をもとに加工・編集して画像を生成する。
多少の脚色は許容する。具体的には:
- 異なるフォルダ（Prosnap / Menu Edit / Kitchen Scene 等）から最適な素材を横断選定
- 構図の大胆なクロップ・リフレーム
- スチームグロウ、光リーク、チルトシフト等のエフェクト追加
- フィルムグレイン、ビネット等の質感演出
- 複数素材のコンポジット

## 単調さの回避
3枚の画像はそれぞれ異なる構図・色味・加工手法を使い、リールで並べた時にストーリー性を持たせる:
- 1枚目: プロセス・臨場感（調理シーン、動きのある演出）
- 2枚目: 主役カット（完成品、ドラマチック照明）
- 3枚目: ディテール（クローズアップ、質感強調）

## 作業手順との関係
「作業手順」にはスタッフが手を動かす工程のみを記載する。
ミセルが自動で済ませた処理（素材選定・AI画像生成）は「使用素材」「AI出力先」に記録し、
「作業手順」には含めない。

# STEP形式ルール（編集手順の書き方）
編集手順は「1 STEP = 1ツール = 1アクション」で書く。
スタッフが上から順にやるだけで完成する。

```
【STEP 1: 写真を選ぶ】
フォルダ: 02_Photos/01_Food/（`1_Pizza` … `7_Cooking_Scene` のカテゴリ別サブフォルダ）
選び方: 例）マルゲリータなら `1_Pizza/` からチーズが見える写真を1枚

【STEP 2: 画像を加工する】
ツール: Google AI Studio → ImageFX
やること: 写真をアップロード → 下のプロンプトをコピペ入力
┌────────────────────────
│ この料理写真を縦9:16にリフレーム。彩度を上げてシズル感を強調。
│ 背景はカフェの暖かい照明
└────────────────────────
→ 生成された画像をダウンロード保存

【STEP 3: 動画にする】
ツール: Google AI Studio → Veo 3.1（※ 上限の場合→DaVinci Resolveのダイナミックズームで代替）
やること: 加工した画像をアップロード → 下のプロンプトをコピペ入力
┌────────────────────────
│ カフェのテーブルに置かれたピザのクローズアップ。
│ チーズがとろける動き、湯気が立ち上る。暖色照明。9:16縦動画
└────────────────────────
→ 生成された動画をダウンロード保存
```

**ルール**:
- AIプロンプトは `┌──` `└──` の罫線枠で囲む（コピペ対象を一目で判別）
- 「or」「または」は禁止（選択肢を作らない）

# 季節自動置換

## ② リールの型（毎本チェック）

**構造（全テーマ共通）**: **動画 7.5秒 → 静止画 2.5秒 × 3**（計15秒）。アルゴリズム上は冒頭の**視聴維持**が最重要。

| 要素 | ルール |
|------|--------|
| **フック** | 動画頭 **0〜2秒** にオンテキスト。Notionの「フック（冒頭3秒）」をそのまま使う |
| **動画** | Veo or 実撮 or 既存.mov。上限時は **DaVinci ダイナミックズーム**で静止画から代替 |
| **画像3枚** | メモのパス or 編集手順のフォルダ。料理とキャプションの一致を意識（`content-prepare` はキャプションキーワードで寄せる） |
| **CTA** | 後半〜最終クリップ付近にオンテキスト。Notionの **CTA** を使用 |
| **トランジション** | クリップ間 **Push 左→右 0.3秒** |
| **BGM** | テンプレ固定。**M'z** = カフェジャズ系 / **Niki** = アップビート系（事前に `04_BGM/` へDL） |
| **書き出し** | 1080×1920、30fps、MP4 |

**Veoの割当**: 1日3回まで → **調理シーン・看板メニュー**優先。空間・ドリンク中心は静止画+ズームで十分。

## ③ KPI・週次改善

**Metaが重視する順（リール）**: **最後まで見た率** → 保存・シェア → いいね・コメント。

| 指標 | Notion列 | 見方・目安 |
|------|----------|------------|
| リーチ | リーチ | 外にどれだけ届いたか |
| 保存 | 保存 | **「また見たい」**の代理指標 → 強い |
| シェア | シェア | 拡散・口コミ |
| いいね / コメント | 各列 | エンゲージメント |
| ER% | ER% | フォロワー比（あれば） |

**アプリで毎回メモする数値（推奨）**: リールの **平均視聴時間** または **視聴維持率**（インサイト画面）— Notionに列が無ければ**メモ欄に1行**で可。

**週次ルーティン（15分）**:
1. 過去7日の投稿を開き、**保存数が多い1本**と**少ない1本**を特定
2. 多い方: フック文・冒頭動画・テーマを **次の企画に再利用**
3. 少い方: **冒頭2秒**（フック・動き・字幕の見え方）だけ仮説を1つ書く（例: 「問いかけが弱い」「料理と文言がズレ」）
4. 翌週の1本だけその仮説を変えて試す（A/Bは週1で十分）

**記録タイミング**: 投稿後 **24〜48時間** くらいでインサイトを見て数値入力（バズり始めたら翌日でも可）。

# Rules
- content-calendar.js のクライアント設定を正として扱う
- ハッシュタグのコピペ繰り返しを避ける（シャドウバンリスク）
- 禁止タグ（#followforfollow 等）は絶対使わない
- 投稿頻度は週3〜5回まで。それ以上はリーチ分散
- 数値の根拠がないテンプレ変更は提案しない
- 新規撮影の予算・時間はない → 既存素材 + AI加工で対応
- Notion rich_text は2000文字制限 → rt() で自動切り詰め
- Veo 3.1 は1日3回制限 → 優先テーマに割り当て
